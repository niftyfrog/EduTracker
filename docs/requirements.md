# EduTracker 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | EduTracker（エデュトラッカー） |
| 目的 | 新人IT研修のカリキュラム管理・進捗可視化 |
| 対象ユーザー | 管理者（教育担当）・新人（研修生） |
| 利用環境 | Webブラウザ（PC） |
| 想定ユーザー数 | ~30名 |

### 1.1 背景・課題

```
【現状の課題】
・新人の研修進捗を管理者が把握しづらい
・カリキュラムの内容がドキュメントに散らばっている
・完了基準が曖昧で、何をもって「終わり」とするか不明確

【EduTrackerで解決すること】
・カリキュラムと進捗を一元管理
・タスクごとの完了条件を明確化（テキスト+画像）
・管理者が全新人の進捗をリアルタイムで把握
```

### 3.3 カリキュラム構成

```
Step 0: Visual Studioのインストール・基本操作    [チェックリスト] 4タスク
  └── インストール / プロジェクト作成 / デバッグ / exe書き出し

Step 1: C# 基礎（コンソールアプリ）             [課題] 6タスク
  ├── 基礎: 変数 / 条件分岐 / ループ / メソッド
  └── 課題: コンソール版電卓 / おみくじ

Step 2: C# + コントロール（Windowsフォーム）    [課題] 2タスク
  └── 課題: GUI版電卓 / GUI版おみくじ

Step 3: SQL                                   [課題] 4タスク
  └── 課題: SELECT / INSERT / UPDATE / DELETE

Step 4〜: 実案件                               [プロジェクト] 動的追加
  └── Java等 / 仕様書ベース
```

### 3.4 タスクのデータ構造

```
タスク
├── id          : 一意識別子
├── no          : 表示番号
├── title       : タイトル
├── tag         : タグ（基礎 / 課題）
├── desc        : 説明文（Markdown）
├── criteria    : 完了条件テキスト（Markdown）
├── criteriaImage: 完了条件の参考画像URL
├── links[]     : 参考リンク配列
└── comments[]  : コメント配列
```

## 4. 非機能要件

### 4.1 性能

| 項目 | 要件 |
|------|------|
| 画面遷移 | 1秒以内 |
| データ読み込み | 3秒以内 |
| 同時利用 | 30ユーザー |
| 画像アップロード | 5MB以下/枚 |

### 4.2 セキュリティ

```
認証・認可:
├── IDベースのログイン認証
├── パスワード: bcryptハッシュ化
├── セッション: JWT（有効期限付き）
├── 権限制御: admin / trainee のロールベース
└── 管理者専用機能のアクセス制御

データ保護:
├── パスワードは平文保存しない
├── API通信は認証トークン必須
└── 外部通信: なし（クローズド環境）
```

### 4.3 可用性・運用

| 項目 | 要件 |
|------|------|
| 稼働時間 | 業務時間内（平日 9:00-18:00） |
| バックアップ | DB: 日次（pg_dump推奨） |
| データ保持 | 無期限 |
| 障害復旧 | Docker再起動で復旧可能 |

### 4.4 対応ブラウザ

| ブラウザ | バージョン |
|---------|-----------|
| Chrome | 最新2バージョン |
| Edge | 最新2バージョン |
| Firefox | 最新2バージョン |

※ モバイル対応: 現時点では対象外（PC利用前提）

## 9. 制約事項・前提条件

### 制約事項

- 外部API通信は行わない（クローズドネットワーク対応）
- AI課題生成機能は本番環境では省略
- モバイル対応は現時点で対象外
- 多言語対応は日本語のみ

### 前提条件

- 客先サーバーでDocker / Docker Composeが利用可能であること
- ブラウザはChrome/Edge/Firefoxの最新版を使用すること
- 社内LANからアクセス可能なサーバーが用意されること

## 10. 用語集

| 用語 | 説明 |
|------|------|
| ステップ (Step) | カリキュラムの大分類。Step 0〜4の5段階 |
| タスク | ステップ内の個別課題。チェックボックスで完了管理 |
| チェックリスト | 確認事項型のステップ（Step 0） |
| 課題 | 成果物提出型のステップ（Step 1〜3） |
| プロジェクト | 実案件型のステップ（Step 4〜） |
| 完了条件 | タスクの合格基準。テキスト+画像で定義 |
| ドロワー | タスクリスト上で右からスライド表示される詳細パネル |
| FAシステムマップ | 社内FAシステムの構成図（参照用） |
